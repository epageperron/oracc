Oracc XML Instance Set (XIS) API Documentation

Glossary Articles

Oracc is organized around the concept of the project, e.g., neo. Projects typically contain a corpus consisting of a catalogue, text editions, and one or more glossaries.

The glossaries are named by ISO639 language codes, e.g., akk.

Glossaries are collections of entries, words or phrases, which have IDs of the form o12345678 or x12345678; these IDs are called OIDs (Oracc IDs). Work to make these OIDs persistent is under way but not yet complete.

Glossary Article URIs

The Glossary Article URI consists of one of each of the above components, in that order. So, for project neo, glossary for language glossary akk, article x00000040, the URI is http://oracc.org/neo/akk/x00000040

.

Glossary Article Instances (XIS)

Each article contains several lists of instances of the words with particular features--the basic meaning, a finer-grained sense, various grammatical parts, etc. These instances are stored as lists whose abbreviation is XIS: XML Instance Sets, so we call the ID for such an instance sets an XIS.

The base URI for an XIS is PROJECT/LANG/XIS: http://oracc.org/neo/akk/akk.r0001d6. This URL returns the first page of results from the XIS.

Query String

Variables in the query string select operations and subsets of the instances returned.

The XIS API has three operations:

page=0
Return the count of instances in the XIS in the XML form <count>[COUNT]</count>. http://oracc.org/neo/akk/akk.r0001d6?page=0.
all
Return all the instances at once. This is a misnomer because the server will only return a maximum of 100 instances. If the number of instances in the XIS is greater than 100 the paging API must be used. http://oracc.org/neo/akk/akk.r0001d6?all.
[PAGE]
Return a page full of instances. http://oracc.org/neo/akk/akk.r0001d6?page=3.

The page size can also be set with size=[ 25 | 50 | 100 ]. Other sizes are not permitted. http://oracc.org/neo/akk/akk.r0001d6?page=2&size=50

The mode in which results are returned can be specified with mode=[ xml | div | html ].

xml
returns the server's XML version of the result set fragment--this is of limited use because the cuneiform transliteration is in Oracc's somewhat technical GDL format. http://oracc.org/neo/akk/akk.r0001d6?page=3&mode=xml
html
returns a formated HTML version of each instance in the set, wrapped in a html. http://oracc.org/neo/akk/akk.r0001d6?page=3&mode=html

Retrieving Documents with Hilited Words

The label part of the instance (the part in parenthesis after the textual instance) contains a word ID of the form P, Q, or X, followed by 6 digits, followed by a period with more digits, then another period with more digits. This is a Word ID. A Word ID can simply be appended to a project URL to display the document with word hilited and scrolled to a reasonable point.
